Setting up a libgdx build environment

07 Dec 2014

In my previous post, 12 Games in 12 Months, I put committed to the lofty goal of shipping 12 games in 12 months. After careful consideration I’ve decided to use the much-celebrated Libgdx framework to help me with my first game. First things first, however, I need to set up my development environment. I prefer to keep my development area as separated as I can from my normal workstation, so will use a virtual machine to house this.

What we’ll need:

  • Ubuntu 14.04.1 Server
  • Desktop Hypervisor (VMWare Workstation, Hyper-V, VirtualBox, etc)

Let’s get started:

  • Install Ubuntu Server 14.04.1.

I know. Right now you may be wondering why I chose the server OS. I did this because I find that the desktop flavor feels a little bloated for what I need. We’re not looking for a full desktop experience.

  • Install a desktop environment.
sudo apt-get update
sudo apt-get install -y --no-install-recommends lubuntu-desktop

This will install a customized lightweight LXDE window manager. I know it’s not the linux way, but I prefer to reboot here. This lets me make sure that the window manager will come up on a reboot and gets the GUI going for me for later.

Now that you’re back up, you’ll need to log in and open a terminal window. Click on the button on the bottom left, go to accessories, then LXTerminal. If there is another terminal you prefer, use that instead.

We will also be installing Google’s Android SDK. Since we’re going for easy, we’ll use Google’s ADT bundle which includes the development IDE, Eclipse, and the Android SDK already married and ready to go.

At the time of writing this, this is the most current link. You may need to update the link below. We’re downloading it, unpacking, then cleaning up after ourselves.

  • Install the SDK
cd ~
wget https://dl.google.com/android/adt/adt-bundle-linux-x86_64-20140702.zip
unzip adt-bundle-linux-x86_64-20140702.zip
rm adt-bundle-linux-x86_64-20140702.zip

Assuming all went well, we’ll have an eclipse folder and an sdk folder under the adt folder. I like to move these to my home folder because I am lazy

optional :

mv adt-bundle-linux-x86_64-20140702/* ~
rmdir mv adt-bundle-linux-x86_64-20140702
  • Next we’ll install Libgdx

First, let’s install the prerequisites.

sudo apt-get install -y libgl1-mesa-dev:i386 openjdk-7-jdk

Next let’s download the gdx-setup.jar

wget http://libgdx.badlogicgames.com/nightlies/dist/gdx-setup.jar

I like to make a folder for my games. Let’s make one and we’ll call it umm.. games.

mkdir ~/games
  • Now, we’ll need to launch eclipse and add the grade integration plugin to eclipse
cd ~/eclipse
./eclipse

Accept the default workspace unless you have a different preference. Click the checkbox to not ask again (unless you like being annoyed– I’m not judging)

I’ll let you pick your usage statistics option. I can’t make all of your choices for you

Next go to Help > Install New Software

You’ll now be greeted by a screen asking for a URL to install your plugin. Since the version of eclipse we are using is < 4.4, we’ll use http://dist.springsource.com/release/TOOLS/gradle under Work With.

You’ll have to name this. I went crazy and named mine Gradle Plugin. You can copy me or pick your own. I’d suggest something memorable…

 

Click OK.  On the next screen, expand the plugins and** choose Gradle IDE**.

On the next screen, select Gradle IDE, then click Next, read the stuff, accept it if you want (can’t move forward without it), then click Finish. Wait for the install.  You’ll be prompted to restart eclipse. Go for it.

Exit Eclipse.

Now list the directories in the home folder.

cd ~
ls

Should look something like this

Now let’s test it all out.

  • We’ll build a project
java -jar gdx-setup.jar

For testing, we can accept the defaults. We will need to specify the path to the SDK and game folder. Make you stuff look something like mine:

Click Generate.

When everything finishes, and you see BUILD SUCCESSFUL, close the Project Generator.

Now we’ll open eclipse again.

cd ~/eclipse
./eclipse

Now we’ll import the libgdx project we created.

File -> Import -> Gradle -> Gradle Project

Click Next, then browse to your game folder.

Click OK.

Now** click Build Model.** Wait..

Click the topmost checkbox to import each project, then click finish.

 

Wait a little more. While you’re waiting, you should see projects start to populate on the left panel in eclipse.

All done.

..

…

Except for that red x next to the android project. What’s that about?

Description Resource Path Location Type
Unable to resolve target ‘android-19′ my-gdx-game-android Unknown Android Target Problem

  • Hm. Looks like we need to fix this.

Go to Window > Android SDK Manager

Wait a few and you should see a lot of options populate. We could add any we want — and you should do this as part of your testing with the emulator later. But not now. Now we’ll just get rid of that error.

Select the 19 SDK option. Try and make sure its the only SDK selected for now. You can leave the defaults of android 5.0 sdk and all that, but you will get prompted when you build asking if you want to use the newer one. Up to you. Also select the API 19.

Click Install XX packages, accept your licenses, and click ok. Now wait some more while the SDK downloads.

Click OK and close the window when you’ve finished. I close and reopen eclipse here. When it’s back up, I give eclipse time to catch up and– what do you know?

The error is gone. Click on the Desktop project and click Run..

Select Java Application and click OK.

Now DesktopLauncher and OK.

Good to go.

And that’s how I set up libgdx from scratch on Ubuntu.
