Anemone : Anonymity as a Service

08 Dec 2014

I’ve been reading about Docker and it’s started me thinking about the possible services that could be hosted. One instance I thought of was hosting the Tor Browser (I’ve since seen people do this as well). Inspired by Slumlord Hosting with Docker which was inspired by Memcached as a Service, I came up with my own recipe.

The Concept

This is the concept, watered down:

You can skip to the bottom for a demonstration video if you’d prefer.

What We’ll Need:

  • Ubuntu 14.04.1 LAMP Server (I got mine from DigitalOcean : Referral Link)
  • Ubuntu 14.04.1 Docker Server

I built these with two network interfaces. One on a private network and one on the public network. It could be argued that the Docker server doesn’t need a public interface.

Log into your Web/LAMP server.

Install LAMP: (Highlights from here):

sudo apt-get update
sudo apt-get install apache2 -y
sudo apt-get install mysql-server libapache2-mod-auth-mysql php5-mysql -y
sudo apt-get install php5 libapache2-mod-php5 php5-mcrypt php5-curl -y
sudo apt-get install makepasswd

sudo mysql_install_db
sudo /usr/bin/mysql_secure_installation
rm /var/www/html/index.html

Install Guacamole and the SQL Authentication Plugin:

sudo apt-get install guacamole-tomcat -y
sudo service tomcat6 stop
sudo service guacd stop

cd ~
wget http://dev.mysql.com/get/Downloads/Connector-J/mysql-connector-java-5.1.23.tar.gz
wget http://downloads.sourceforge.net/project/guacamole/current/extensions/guacamole-auth-mysql-0.8.4.tar.gz

tar xzvf guacamole-auth-mysql-0.8.4.tar.gz
tar xzvf mysql-connector-java-5.1.23.tar.gz

sudo mkdir /var/lib/guacamole/classpath

sudo cp guacamole-auth-mysql-0.8.4/lib/* /var/lib/guacamole/classpath/
sudo cp mysql-connector-java-5.1.23/mysql-connector-java-5.1.23-bin.jar /var/lib/guacamole/classpath/

mysql -u root -p
CREATE DATABASE guacamole;
CREATE USER 'guacamole'@'localhost' IDENTIFIED BY 'YOURPWCHANGEME!';
GRANT SELECT,INSERT,UPDATE,DELETE ON guacamole.* TO 'guacamole'@'localhost';
FLUSH PRIVILEGES;
exit

cat guacamole-auth-mysql-0.8.4/schema/*.sql | mysql -u root -p guacamole

sudo nano /etc/guacamole/guacamole.properties

replace the contents with this:

guacd-hostname: localhost
guacd-port: 4822

lib-directory: /var/lib/guacamole/classpath

# Auth provider class
auth-provider: net.sourceforge.guacamole.net.auth.mysql.MySQLAuthenticationProvider

# MySQL properties
mysql-hostname: localhost
mysql-port: 3306
mysql-database: guacamole
mysql-username: guacamole
mysql-password: YOURPWCHANGEME!

sudo service tomcat6 start
sudo service guacd start

You should be able to hit http://yourip:8080/guacamole now. The default username:pass is guacadmin:guacadmin. I suggest you change that now…

Log into your Docker server.

Install Docker (Highlights from here):

sudo apt-get update
sudo apt-get install linux-image-extra-`uname -r` -y
sudo sh -c "wget -qO- https://get.docker.io/gpg | apt-key add -"
sudo sh -c "echo deb http://get.docker.io/ubuntu docker main\
> /etc/apt/sources.list.d/docker.list"
sudo apt-get update
sudo apt-get install lxc-docker -y

Create your Dockerfile

nano Dockerfile

from ubuntu:trusty
run apt-get update
run apt-get install -y wget x11vnc xvfb xz-utils firefox
run useradd anon --create-home -s /bin/sh
run wget https://dist.torproject.org/torbrowser/4.0.2/tor-browser-linux64-4.0.2_en-US.tar.xz
run tar -C /home/anon -xvJf tor-browser-linux64-4.0.2_en-US.tar.xz
run rm /*.xz

add start.sh /home/anon/start.sh

run chmod +x /home/anon/start.sh

cmd ["/home/anon/start.sh"]

EXPOSE 5900

USER anon

Create the start.sh

nano start.sh

#!/bin/bash
x11vnc -once -create -geometry 1280x1024 \
-env X11VNC_CREATE_GEOM=${1:-1280x1024x8} \
-env FD_PROG="/home/anon/tor-browser_en-US/start-tor-browser \
-height 1024 -width 1280" &

sleep 10m

Build the docker image:

sudo docker build -t anemone_img .

Enable the docker remote API **(from here): **

**There is no authentication for the Docker the API out of the box. You should implement your own. I am restricting the API to the private IP address. In a real world deployment you should implement as many security best practices as you feasibly can. This can include changing the default port, listenting only on the private interface, implementing authentication, SSL, etc.**

sudo nano /etc/init/docker.conf

Find the line that starts with DOCKER_OPTS, and change it to the below

DOCKER_OPTS='-H tcp://YOURDOCKER'sPRIVATEIP:4243 -H unix:///var/run/docker.sock'

Save, Exit. Then run:

sudo service docker stop
sudo service docker start

Log back into your webserver.

Please be understanding. I hack things together. I never said I do it well– or prettily. I know I should be using stored procedures. I know I am not using best practices. I know my PHP sucks. Instead of tearing me up for writing bad code, create your own and share?

session_start();

unset($_SESSION['UN']);
unset($_SESSION['PW']);

}
}

if(!isset($_SESSION['started']))

$ch = curl_init('http://127.0.0.1/createInstance.php');

createInstance.php

<?php
$dockerhost = 'YOUR DOCKER HOST'S PRIVATE IP';
$data_string = '{"Image":"anemone_img"}';

$ch = curl_init('http://'.$dockerhost.':4243/containers/create');
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($ch, CURLOPT_POSTFIELDS, $data_string);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, array(
    'Content-Type: application/json',
    'Content-Length: ' . strlen($data_string))
);

$result = curl_exec($ch);

if( !($result = curl_exec($ch)) ) {
    curl_close($ch);
    exit;
}
curl_close($ch);
$obj = json_decode($result);

$contId = $obj->{'Id'};

$rndPort = mt_rand(11000, 11900);
$data_string = '{"PortBindings":{ "5900/tcp": [{ "HostIp" : "'.$dockerhost.'", "HostPort": "'.$rndPort.'" }] }}';

$ch = curl_init('http://'.$dockerhost.':4243/containers/' . $contId . '/start');
curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($ch, CURLOPT_POSTFIELDS, $data_string);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, array(
    'Content-Type: application/json',
    'Content-Length: ' . strlen($data_string))
);

$result = curl_exec($ch);

curl_close($ch);

$ch = curl_init('http://'.$dockerhost.':4243/containers/' . $contId . '/json');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$result = curl_exec($ch);
if( !($result = curl_exec($ch)) ) {
    curl_close($ch);
    exit;
}
curl_close($ch);

$obj = json_decode($result);

$hostPort =  $obj->HostConfig->{'PortBindings'}->{'5900/tcp'}[0]->{'HostPort'};
$name = $obj->Name;

curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
$result = curl_exec($ch);
$arr = array ('name'=>$name,'pw'=>$pass);
echo json_encode($arr);

?>

Setting up a libgdx build environment

07 Dec 2014

In my previous post, 12 Games in 12 Months, I put committed to the lofty goal of shipping 12 games in 12 months. After careful consideration I’ve decided to use the much-celebrated Libgdx framework to help me with my first game. First things first, however, I need to set up my development environment. I prefer to keep my development area as separated as I can from my normal workstation, so will use a virtual machine to house this.

What we’ll need:

  • Ubuntu 14.04.1 Server
  • Desktop Hypervisor (VMWare Workstation, Hyper-V, VirtualBox, etc)

Let’s get started:

  • Install Ubuntu Server 14.04.1.

I know. Right now you may be wondering why I chose the server OS. I did this because I find that the desktop flavor feels a little bloated for what I need. We’re not looking for a full desktop experience.

  • Install a desktop environment.
sudo apt-get update
sudo apt-get install -y --no-install-recommends lubuntu-desktop

This will install a customized lightweight LXDE window manager. I know it’s not the linux way, but I prefer to reboot here. This lets me make sure that the window manager will come up on a reboot and gets the GUI going for me for later.

Now that you’re back up, you’ll need to log in and open a terminal window. Click on the button on the bottom left, go to accessories, then LXTerminal. If there is another terminal you prefer, use that instead.

We will also be installing Google’s Android SDK. Since we’re going for easy, we’ll use Google’s ADT bundle which includes the development IDE, Eclipse, and the Android SDK already married and ready to go.

At the time of writing this, this is the most current link. You may need to update the link below. We’re downloading it, unpacking, then cleaning up after ourselves.

  • Install the SDK
cd ~
wget https://dl.google.com/android/adt/adt-bundle-linux-x86_64-20140702.zip
unzip adt-bundle-linux-x86_64-20140702.zip
rm adt-bundle-linux-x86_64-20140702.zip

Assuming all went well, we’ll have an eclipse folder and an sdk folder under the adt folder. I like to move these to my home folder because I am lazy

optional :

mv adt-bundle-linux-x86_64-20140702/* ~
rmdir mv adt-bundle-linux-x86_64-20140702
  • Next we’ll install Libgdx

First, let’s install the prerequisites.

sudo apt-get install -y libgl1-mesa-dev:i386 openjdk-7-jdk

Next let’s download the gdx-setup.jar

wget http://libgdx.badlogicgames.com/nightlies/dist/gdx-setup.jar

I like to make a folder for my games. Let’s make one and we’ll call it umm.. games.

mkdir ~/games
  • Now, we’ll need to launch eclipse and add the grade integration plugin to eclipse
cd ~/eclipse
./eclipse

Accept the default workspace unless you have a different preference. Click the checkbox to not ask again (unless you like being annoyed– I’m not judging)

I’ll let you pick your usage statistics option. I can’t make all of your choices for you

Next go to Help > Install New Software

You’ll now be greeted by a screen asking for a URL to install your plugin. Since the version of eclipse we are using is < 4.4, we’ll use http://dist.springsource.com/release/TOOLS/gradle under Work With.

You’ll have to name this. I went crazy and named mine Gradle Plugin. You can copy me or pick your own. I’d suggest something memorable…

 

Click OK.  On the next screen, expand the plugins and** choose Gradle IDE**.

On the next screen, select Gradle IDE, then click Next, read the stuff, accept it if you want (can’t move forward without it), then click Finish. Wait for the install.  You’ll be prompted to restart eclipse. Go for it.

Exit Eclipse.

Now list the directories in the home folder.

cd ~
ls

Should look something like this

Now let’s test it all out.

  • We’ll build a project
java -jar gdx-setup.jar

For testing, we can accept the defaults. We will need to specify the path to the SDK and game folder. Make you stuff look something like mine:

Click Generate.

When everything finishes, and you see BUILD SUCCESSFUL, close the Project Generator.

Now we’ll open eclipse again.

cd ~/eclipse
./eclipse

Now we’ll import the libgdx project we created.

File -> Import -> Gradle -> Gradle Project

Click Next, then browse to your game folder.

Click OK.

Now** click Build Model.** Wait..

Click the topmost checkbox to import each project, then click finish.

 

Wait a little more. While you’re waiting, you should see projects start to populate on the left panel in eclipse.

All done.

..

…

Except for that red x next to the android project. What’s that about?

Description Resource Path Location Type
Unable to resolve target ‘android-19′ my-gdx-game-android Unknown Android Target Problem

  • Hm. Looks like we need to fix this.

Go to Window > Android SDK Manager

Wait a few and you should see a lot of options populate. We could add any we want — and you should do this as part of your testing with the emulator later. But not now. Now we’ll just get rid of that error.

Select the 19 SDK option. Try and make sure its the only SDK selected for now. You can leave the defaults of android 5.0 sdk and all that, but you will get prompted when you build asking if you want to use the newer one. Up to you. Also select the API 19.

Click Install XX packages, accept your licenses, and click ok. Now wait some more while the SDK downloads.

Click OK and close the window when you’ve finished. I close and reopen eclipse here. When it’s back up, I give eclipse time to catch up and– what do you know?

The error is gone. Click on the Desktop project and click Run..

Select Java Application and click OK.

Now DesktopLauncher and OK.

Good to go.

And that’s how I set up libgdx from scratch on Ubuntu.

I never finish my games. I start them. My wife tells me I have some form of A.D.D. I can almost agree with her, but from my perspective it’s like seeing something shiny and new and wanting to play with that for awhile. I always mean to come back and finish …

I need to finish my games.

So, I’ve been reading a bit — and I see that I’ve not been the only person afflicted with this condition– technology induced A.D.D.

Reading posts by folks like levelsIO, I had this amazing idea that I would complete 12 games in 12 months. I then found out I was about 2 years late to the party.

I am not deterred– in fact, I am doubly motivated and I’ll give it a shot anyway.

I will rate my games– and post my progress here to everyone for accountability.

Multipliers:

  • Multiplatform : x multiplier (x being the number of supported platforms)
  • Multiplayer : x multiplier (x being the number of supported players)

Wish me luck!
